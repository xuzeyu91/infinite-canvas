# 待测试

- 画布生成前的 API Key 提示：当画布里触发任意依赖模型的生成行为且当前未配置可用 API Key 时，会自动弹出“配置与用户偏好”弹窗，自动切到“渠道”页，并提示需要先填写 API Key 后才能继续使用生成功能。
- 画布提示词面板新增 `AI优化`：在节点 Prompt Panel 里新增 `AI优化` 按钮，点击后会基于当前生成模式（文本/图片/视频/音频）、节点现有内容状态和已连接的场景参考（图片/视频/音频/文本）自动重写提示词；优化结果默认控制在较短范围，避免“小说式”冗长描述，并自动回填到输入框。文本节点单击也会和图片/视频节点一样直接弹出该面板（不再必须先点“编辑文字”）。
- 渠道配置入口调整：移除“新增渠道”、单渠道“拉取模型”和“删除渠道”功能；在渠道的 `API Key` 输入框右侧新增“去获取”按钮，点击后新窗口跳转到 `https://api.antsk.cn`。
- 画布视频参考图上限提示：视频节点左上角新增“参考图 x/y”计数；超过当前视频模型支持上限时显示红色“超出”状态；拖拽连线若导致任一视频生成节点超限会阻止连接并提示，视频生成或重试前也会给出相同提示。
- 提示词卡片封面兜底：`PromptCard` 在 `coverUrl` 为空字符串时不再渲染空 `img src`，改为展示统一占位背景，避免浏览器控制台出现 “An empty string ("") was passed to the src attribute” 报错。
- 画布模型记忆优化：生成配置节点与节点提示面板的模型选择改为按模式独立记忆（生图/文本/视频/音频），切换模式后会恢复该模式上次选择的模型，不再被其他模式覆盖。
- 工作台入口调整：顶部导航和移动端导航中的“生图工作台”“视频创作台”改为外链，分别跳转到 `https://bigbanana.tree456.com/gemini-image.html` 与 `https://bigbanana.tree456.com/ai-video-content.html`；站内 `/image`、`/video` 页面已移除。
- 配置与用户偏好：渠道 Base URL 固定为 `https://api.antsk.cn` 且不可编辑；默认文本模型更新为 `gpt-5.4`、默认音频模型更新为 `gpt-audio-1.5`，图片模型新增 `gemini-3-pro-image-preview`，视频模型默认可选项更新为 `sora-2`、`veo_3_1-fast`、`viduq3-turbo`、`viduq3-pro`、`doubao-seedance-1-5-pro`、`doubao-seedance-2-0-fast`、`doubao-seedance-2-0`。
- 配置与用户偏好（模型配置简化）：渠道页移除“拉取全部”按钮和左侧提示说明，模型列表改为手动维护；默认模型列表参考 `BigBanana-AI-Director` 补齐，文本模型新增 `gpt-5.5`、`gpt-5.2`、`gpt-5.1`、`claude-sonnet-4-6`、`claude-opus-4-6`、`claude-opus-4-7`、`gemini-3.1-pro-preview`，图片模型新增 `gemini-3.1-flash-image-preview`、`gpt-image-1.5`，视频模型新增 `happyhorse-1.0`，音频模型新增 `gpt-audio-mini`。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI 兼容渠道。
- 请求格式对齐（参考 BigBanana-AI-Director）：文本问答与在线 Agent 工具调用改为 `v1/chat/completions`，`gpt-audio-1.5` 默认改为 `v1/chat/completions` 音频输出格式；`viduq3-*` 改为 `v1/videos` JSON 网关参数，`sora-2`、`veo_3_1-fast`、`doubao-seedance-*` 使用 `v1/videos` 异步任务格式，火山任务模型仍走 `/contents/generations/tasks`。
- Gemini 生图修复：`gemini-3-pro-image-preview` 按模型名自动走 Gemini 请求格式（避免误走 OpenAI images 接口）；请求体补齐 `generationConfig.imageConfig.aspectRatio`，与参考项目保持一致。
- 画布 UI 优化：将节点配置/生成面板（Prompt Panel）的宽度从 `500px` 增加到 `580px`，并将模型选择器（ModelPicker）的默认高度统一为 `h-10`（40px），最大宽度限制为 `180px`，提示词库按钮也统一为 `h-10` 尺寸，彻底解决长模型名称（如 `gemini-2-pro-image-prev...`）导致右侧设置按钮与生成按钮重叠、挤压的问题。
- 页面样式与配色优化（参考 BigBanana-AI-Director）：将画布的亮色与暗色主题配色进行调整。亮色主题采用 Claude 风格的暖色调（背景 `#F7F5F0`，主文字 `#1A1816`，辅文字 `#44413D`，暖棕色 `#B5936B` 强调色），暗色主题采用极简纯黑风格（背景 `#050505`，主文字 `#ffffff`，辅文字 `#d4d4d8`，靛蓝色 `#818cf8` 强调色）。同时将 Ant Design 全局组件的主题背景、边框 and 文字颜色与之对齐，使整体视觉风格更加和谐统一。
- 画布库卡片样式优化（参考 BigBanana-AI-Director）：将画布列表中的卡片组件替换为 Ant Design 的 `<Card hoverable>` 组件，使其自动继承全局配置的边框颜色、圆角、背景色以及精致的悬停阴影、悬停边框过渡等效果。这使画布卡片与提示词卡片（PromptCard）的视觉风格、边框质感和交互反馈达到 100% 完美一致，极大提升了界面的整体感和精致度。
- 画布节点激活边框优化（参考 BigBanana-AI-Director）：将画布中被选中或激活的节点（Node）的蓝色边框（`#2f80ff`）替换为与当前主题高度契合的强调色（亮色模式下为 Claude 暖棕色 `#B5936B`，暗色模式下为靛蓝色 `#818cf8`）。同时，图片组展开数量、设为主图的星星图标、资源标签徽章（ResourceLabelBadge）以及提示词文本中的提及高亮（MentionHighlightText）也同步对齐为该主题强调色，彻底消除了蓝色在暖色/纯黑主题下的突兀感，使画布内的交互高亮更加和谐自然。
- 进入画布卡顿优化：在“我的画布”页增加首个项目的空闲预取，并在卡片 hover/focus/touch 时按需预取目标 `canvas/[id]` 路由；同时将画布页初始化时的节点恢复与助手图片恢复改为并行执行，减少点击卡片进入画布时的首屏等待。
